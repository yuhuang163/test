# 代码开发流程

## 常规开发流程

1. 切到 `develop` 并更新

```bash
git checkout develop
git pull
```

2. 拉新分支

```bash
git checkout -b feature/xxx
```

3. 开发并提交

```bash
git add .
git commit -m "feat: xxx"
```

4. 推送分支

```bash
git push origin feature/xxx
```

5. 在 GitLab 上发起合并请求（MR）

## 解冲突流程

为避免冲突，建议开发过程中持续同步 `develop`。

1. 切回 `develop`

```bash
git checkout develop
git pull
```

2. 回到你的分支

```bash
git checkout feature/xxx
```

3. 合并 `develop`

```bash
git merge develop
```
