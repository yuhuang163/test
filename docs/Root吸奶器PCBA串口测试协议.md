# UART通讯标准

测试指令通过标准串口通讯，串口波特率1000000，数据位8，校验位none，停止位1。

设备开机后会有如下打印信息，表示设备正常完成初始化，可以开始接收测试指令：例如“----------LutePump\[01\]V2.1----------”。（其中“LutePump”为固定，“\[01\]”表示设备型号，“V2.1”表示软件版本号）

# 1软件数据传输协议

## 数据字节顺序

16位/32位的数据均采用小端模式传输。

| unsigned char (UINT8) | unsigned short(UINT16) |  | unsigned int(UINT32)/Float |  |  |  |
| --- | --- | --- | --- | --- | --- | --- |
| BYTE | BYTE0(LSB) | BYTE1(MSB) | BYTE0(LSB) | BYTE1 | BYTE2 | BYTE3(MSB) |

## 数据包格式介绍

一个完整的数据包格式定义如下：

| Protocol Packet Structure |  |  |  |  |  |  |
| --- | --- | --- | --- | --- | --- | --- |
| Start Of Packet<br>(SOP) | Frame Control Field<br>(FCF) | Command Type（CT） | Command ID(CID) | Command Argument（CA） |  | CheckSum8 |
|  |  |  |  | Command Argument Len（CAL） | Command Argument Body（CAB） |
| U8 | U8 | U8 | U8 | U8 | Variable | U8 |
| 0xAA | 0x55 | 0x00(REQ) | — | — | — | — |
|  |  | 0x01(ACK) |
|  |  | 0x02(NACK) |
|  |  | 0x03(Notify) |

1.  帧头SOP+FCF：0xAA 0x55
    
2.  命令类型（CT）描述：
    
    REQ：数据请求，需有ACK或者NACK应答；
    
    ACK：数据接收后正确应答，返回REQ所需数据；
    
    NACK：数据接收否定应答，返回错误信息；
    
    Notify：用于设备自动反馈数据，不需要ACK或NACK应答。
    
3.  命令（CID）：详见命令指令ID部分。
    
4.  命令参数长度（CAL）：1个字节，最大参数长度20字节。
    
5.  命令参数数据（CAB）：字节数量等于CAL。
    
6.  CheckSum8计算方法：数据包除校验字节本身外的所有数据按字节求和取反。（前期调试可以使用0xFF代替）参考代码如下：
    

u8 check\_sum(u8 \*dat, u8 len)

{

    u8 i = 0;

u32 sum = 0;

for (i = 0; i < len; i++)

{

sum += dat\[i\];

}

    return ~(u8)sum;

}
