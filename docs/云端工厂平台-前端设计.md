# 云端工厂平台 — 前端设计（网页管理端）

> **工厂云 Web 管理门户** UI/UX 与页面设计。  
> 面向工艺、质量、运维；**不替代**产测上位机 Qt 界面。  
> 接口契约见 [云端工厂平台-后端设计.md](./云端工厂平台-后端设计.md)；字段细节见 [云端工厂平台功能设计.md](./云端工厂平台功能设计.md)。

**范围说明**：本门户 **不包含** MES 过站、过站 JSON、测试结果报表查询；过站与 MES 仍由各厂 `qmes` 与上位机本地 `test_pass_data.db` / CSV 处理。**工厂云网页**聚焦：日志、阈值、用例、上位机 OTA、账号与设备管理。

## 2. 目标用户与角色

| 角色 | 典型用户 | 主要页面 |
|------|----------|----------|
| **operator** | 产线班组长 | 日志查询（只读） |
| **engineer** | 工艺工程师 | 阈值模板、测试用例、发布记录 |
| **admin** | 运维 / IT | 账号管理、上位机版本、设备登记、全部 engineer 能力 |

菜单与按钮按角色 **动态隐藏**；无权限 API 返回 403 时统一 Toast 提示。

## 3. 技术栈（已定）

| 项 | 选型 |
|----|------|
| 语言 | **JavaScript**（ES2020+，**不用 TypeScript**） |
| 框架 | **Vue 3**（Composition API + `<script setup>`） |
| 构建 | Vite |
| UI | Element Plus / Ant Design Vue |
| 路由 | Vue Router 4 |
| 状态 | Pinia（user、token、station 字典） |
| HTTP | axios；拦截器注入 `Authorization: Bearer` |
| 接口文档 | 对接后端 FastAPI OpenAPI；可用 JSDoc `@typedef` 手写或 openapi 工具生成 **`.js` 注释**（可选） |

**部署**：静态资源挂 `https://fctp.luteos.com/admin/` 或独立子域 `https://factory-admin.luteos.com`；构建产物为纯静态文件，由 Nginx 托管，业务 API 反代到 **Python FastAPI** 服务。

### 3.1 HTTP 与文件传输

与总览 **§4.3**、[后端 §5.4](./云端工厂平台-后端设计.md) 一致：

| 场景 | 前端做法 |
|------|----------|
| 登录、CRUD、列表 | axios + `application/json` |
| 上传 exe（小文件） | `FormData` + `multipart/form-data` |
| 上传 exe（大文件，推荐） | 先 `POST /admin/host-app/upload-url` 得预签名 URL，再 **PUT 二进制到 OSS**，最后 JSON 登记元数据 |
| 下载日志 / bundle | axios `responseType: 'blob'` 触发浏览器保存，或打开后端返回的签名 URL |

**注意**：下载接口响应为二进制，**不要**按 `{ code, message, data }` 解析；仅 JSON API 走统一包络。

- **工站筛选**：顶栏全局 `stationKey`（ engineer/admin 可选「全部」）；operator 仅见授权工站。

- **未登录**：除 `/login` 外一律重定向登录页。

## 6. 页面设计

### 6.1 登录 `/login`

| 元素 | 说明 |
|------|------|
| 用户名 / 密码 | 工厂云账号 |
| 登录按钮 | `POST /auth/login`；网页 **不传 hostName** 或传 `web-console` 作为 clientType |
| 错误提示 | 账号停用、锁定、工站未授权等展示 `message` |

登录成功：存 `accessToken`、`roles`、`stationKeys` → 跳转 `/dashboard` 或上次路径。

> 说明：上位机 login **必须** `hostName`（电脑名）；网页登录可用固定 `clientType: "web"` 区分审计（后端扩展字段，可选）。

### 6.2 日志查询 `/data/logs`

**列表**：**工厂**（`factoryDisplayName`）、上传时间、工站、deviceId、hostName、SN（若有）、testResult（若有）、包大小、操作（查看 / 下载）。

**筛选**：**工厂**、时间、工站、SN、deviceId、hostName。工厂下拉数据来自 `GET /admin/meta/factories`。

**详情页** `/data/logs/{id}`：

| 区域 | 说明 |
|------|------|
| 顶栏元数据 | 工厂、工站、deviceId、hostName、上传时间、SN、testResult |
| 左 | 解压后文件树（`上位机log/*.txt` 等） |
| 右 | 选中 `.txt` 在线预览（`GET /logs/{id}/files/{path}`）；支持搜索、滚动 |
| 操作 | 「下载完整 zip」 |

**下载**：`GET /logs/{id}/download`（原 zip）。

### 6.3 阈值模板 `/config/thresholds`

**列表**：模板名、工站、产品型号、当前已发布 version、更新时间、状态（草稿/已发布）。

**编辑页**

| 区域 | 说明 |
|------|------|
| 基本信息 | 名称、stationKey、productModel |
| 键值表 | `settingsKey` 下拉（来自 `/admin/meta/settings-keys` 白名单）+ value 输入 |
| 校验 | 键名必填、数值型 RSSI/电流范围提示 |
| 操作 | 保存草稿、`发布`（生成新 version，确认弹窗） |

**发布记录**：历史 version 只读查看，支持对比（二期）。

### 6.4 测试用例 `/config/test-cases`

**布局**：左文件树 + 右编辑器（ini 文本）+ 底流程预览（可选）。

| 功能 | 说明 |
|------|------|
| 文件树 | `test_case/` 下 ini + `总的测试流程.ini` |
| 编辑器 | 语法高亮；保存 `PUT /admin/test-cases/files/{path}` |
| 流程 Tab | 按 `[Station/xxx] Items=` 可视化编排（对接上位机 `test_flow_editor` 字段模型，二期） |
| 发布 | `发布 bundle` → 升 `bundleVersion`，打 zip |
| 校验 | 发布前检查路径非法字符、重复 case 名 |

**参考上位机模型**：`platform/test_case/test_case.h`、`test_case_edit_dialog`（只读对照，不要求 Web 一期全字段）。

### 6.5 上位机版本 `/config/host-app`

**列表**：appVersion、buildId、packageName、发布时间、forceUpgrade、灰度范围、操作。

**新建版本向导**

1. 上传 exe（或填 OSS 已上传 URL）  
2. 自动计算 sha256（前端 Web Crypto 或后端上传后返回）  
3. 填写 releaseNotes、appVersion、buildId  
4. 灰度：多选 stationKey、factory、deviceId 白名单  
5. 勾选 forceUpgrade  
6. 提交 `POST /admin/host-app/versions`

### 6.6 统一发布 `/config/releases`（建议 P2）

**目的**：一次发布勾选对外的阈值 / 用例 / exe 组合，避免现场不知道更了啥。

| 步骤 | UI |
|------|-----|
| 新建发布单 | 填写 releaseId、说明 |
| 选择产物 | 下拉选已发布的 threshold version、bundle version、host buildId（均可选） |
| 灰度 | 同 OTA |
| 发布 | 写入 `releases`；上位机 `releases/check` 一次返回 |

**列表**：releaseId、包含层级标签、时间、操作人。

### 6.7 账号管理 `/system/users`（admin）

| 功能 | UI |
|------|-----|
| 列表 | 用户名、角色、工站、状态、最后登录时间、最后登录 hostName |
| 新建 | 用户名、初始密码（仅此次展示）、角色、stationKeys 多选、备注 |
| 编辑 | 角色、工站、启用/停用 |
| 重置密码 | 二次确认；生成随机密码弹窗复制 |
| 锁定 | 显示 failed_login_count；手动解锁按钮 |

策略见总览 **§5.6.3～§5.6.8**。

### 6.8 设备登记 `/system/devices`（admin）

登记产线 PC：`hostName`（电脑名）、产线、工位、备注。  
列表支持按 hostName 搜索；与登录审计联动，未登记 PC 登录高亮（二期告警）。

### 6.9 登录审计 `/system/audit-logins`（admin）

表格：时间、用户名、hostName、deviceId、stationKey、action（login/logout/fail）、IP。  
筛选：时间、用户、hostName。

### 6.10 个人中心 `/profile`

修改密码（旧密码 + 新密码）；退出登录。

## 7. 交互与体验规范

| 项 | 规范 |
|----|------|
| 语言 | 简体中文 |
| 时间 | 显示 `yyyy-MM-dd HH:mm:ss` |

## 8. 与上位机的分工

| 能力 | 上位机 Qt | 网页 |
|------|-----------|------|
| 产测执行 | ✓ | — |
| MES 过站 | ✓（`qmes`，不在工厂云） | — |
| 工厂云登录 | ✓（带 hostName） | ✓（web client） |
| 上传日志 | ✓ | 查询、下载 |
| 编辑 test_case | 可选本地 | **主** |
| 编辑阈值 | 只读拉取 | **主** |
| 发版 exe | 只 check/下载 | **主** |
