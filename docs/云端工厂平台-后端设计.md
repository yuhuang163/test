# 云端工厂平台 — 后端设计

> **工厂云服务端**（`fctp.luteos.com` / `api/factory-tool/*`）技术方案。  
> 与上位机 Qt、网页管理端契约以本文 + [云端工厂平台功能设计.md](./云端工厂平台功能设计.md) 为准。

**范围说明**：本服务 **不提供** MES 过站对接、过站 JSON 上报（`/test-records`）及过站报表；各厂 MES 仍走上位机 `agreement/qmes/*`。**工厂云后端**聚焦：登录、日志归档、阈值/用例下发、上位机 OTA、账号与审计。

## 2. 目标与边界

### 2.1 职责

- 为产测 **上位机 Qt** 提供：登录、日志归档、阈值/用例下发、上位机 exe OTA 元数据与下载。
- 为 **网页管理端** 提供：账号 RBAC、阈值/用例/版本编辑与发布、日志查询与下载、审计。
- **对象存储**：zip 日志、test_case 包、exe 安装包。

### 2.2 不在本服务内

- **各厂 MES 专网对接、过站结果上云、过站报表**（`test-records` 等）
- 设备固件 OTA

## 3. 技术栈（已定）

| 层次 | 选型 | 说明 |
|------|------|------|
| 语言 | **Python 3.11+** | 与上位机 Qt/C++ 分离，独立部署 |
| 框架 | **FastAPI** | REST + 自动生成 OpenAPI 3.0；ASGI 部署用 **Uvicorn**（生产可加 Gunicorn workers） |
| ORM / 迁移 | SQLAlchemy 2 + Alembic | 或 Tortoise ORM（团队习惯二选一） |
| 鉴权 | python-jose / PyJWT | Bearer Token；与网页、上位机共用契约 |
| 数据库 | PostgreSQL 或 MySQL 8 | 业务与审计 |
| 缓存 | Redis | Token 黑名单、登录锁定计数、manifest 缓存 |
| 对象存储 | MinIO / 阿里云 OSS / S3 | 大文件；DB 只存元数据与 URL |
| 文档 | OpenAPI 3.0 | FastAPI 自带 `/docs`；导出 yaml 供前端对齐 |
| 依赖管理 | pip + `requirements.txt` 或 **uv** / Poetry | 锁定版本便于 CI |

**备选**：若更偏「内置 Admin、复杂 RBAC 表单」，可改用 **Django + Django REST framework**；API 路径与 JSON 包络仍按本文，不与 FastAPI 混用两套服务。

**域名**：`https://fctp.luteos.com`  
**API 前缀**：`/api/factory-tool`

```text
  上位机 Qt ────────►│  API 网关   │◄────│  网页前端   │
                    │  HTTPS      │     │ Vue3 + JS   │
                    └──────┬──────┘     └─────────────┘
                           │
         ┌─────────────────┼─────────────────┐
         ▼                 ▼                 ▼
   auth-service      business-api      file-service
   登录/Token/RBAC    阈值/用例/OTA      上传/签名 URL
         │                 │                 │
         └────────┬────────┴────────┬────────┘
                  ▼                 ▼
            PostgreSQL           对象存储
```

### 4.1 模块划分（逻辑服务，可单体起步）

| 模块 | 职责 |
|------|------|
| **auth** | login / refresh / logout / me；用户 CRUD；角色与 `stationKeys`；登录审计 |
| **logs** | multipart 接收 zip（含 factoryName）；解压索引；元数据入库；查询、预览与下载 |
| **thresholds** | 阈值模板版本、发布、按 `stationKey`+`productModel` 拉取 |
| **test-cases** | manifest、bundle zip、单文件；版本与 sha256 |
| **host-app** | exe 版本、buildId、灰度规则、check、download 重定向或流式 |
| **releases**（建议） | 统一发布单：一次发布可含 threshold + bundle + exe 三层版本（见 §8） |
| **admin** | 工站字典、已登记 PC、系统配置 |

## 5. 公共约定

与总览 **§4.1** 一致，后端须强制：

### 5.1 响应包络

```json
{ "code": 0, "message": "ok", "data": {} }
```

| code | HTTP | 含义 |
|------|------|------|
| 0 | 200 | 成功 |
| 401 | 401 | 未登录 / Token 无效 |
| 403 | 403 | 无权限 / 工站未授权 |
| 404 | 404 | 资源不存在 |
| 409 | 409 | 版本冲突 |
| 5xx | 5xx | 服务端错误 |

### 5.2 请求头（上位机 / 网页）

| 头 | 必填 | 说明 |
|----|------|------|
| `Authorization` | 除 login、过渡期 logs 外 | `Bearer <accessToken>` |
| `Device-Id` | 上位机必填 | 设备标识，默认同电脑名 |
| `Station-Key` | 建议 | 当前工站 |
| `APP-Version` | 建议 | 上位机语义版本 |
| `X-Client-Build` | 可选 | buildId |

### 5.3 鉴权模型

- 密码 **bcrypt** 存储。
- 角色：`operator` | `engineer` | `admin`（见总览 §5.6.2）。
- 用户绑定 `station_keys[]`；接口按角色 + 工站校验。
- **login 必填 `hostName`**（电脑名）；写入会话与审计表。
- Token：`accessToken` 8～12h；`refreshToken` 7～30d（二期）；Redis 维护注销与单 PC 单会话（可选）。

账号策略详见总览 **§5.6.3～§5.6.8**。

### 5.4 通信方式与 Content-Type（文件传输）

与总览 **§4.3** 一致；后端实现须严格区分：

| 类型 | Content-Type | 响应包络 |
|------|----------------|----------|
| 控制面 / 元数据 | `application/json` | `{ code, message, data }` |
| 上传文件 | `multipart/form-data` | 响应 JSON |
| 下载文件 | `application/octet-stream` / `application/zip` | **无 JSON 包络**，直接字节流；或 **302** 到 OSS |

**文件类接口一览**：

| 接口 | 上传/下载 | 存储 |
|------|-----------|------|
| `POST /logs/upload` | 上传 zip | OSS `logs/{factory_name}/...` |
| `GET /logs/{id}/download` | 下载 zip | 流式或签名 URL |
| `GET /test-cases/bundle` | 下载 zip | OSS `test-cases/bundles/` |
| `GET /test-cases/{name}` | 下载单 ini | `text/plain` 或 `application/octet-stream` |
| `GET /host-app/download/{buildId}` | 下载 exe | OSS `host-app/` |
| `POST /admin/host-app/versions` | 上传 exe（multipart）或 JSON 仅元数据 | OSS |

**管理端发版 exe 推荐流程**（大文件）：

```text
1. POST /admin/host-app/upload-url  → 返回 { uploadUrl, objectKey }  （可选预签名 PUT）
2. 网页 PUT 二进制到 OSS
3. POST /admin/host-app/versions    → JSON：buildId, sha256, appVersion, grayRules...
```

**sha256**：服务端在入库前对 OSS 对象计算或信任客户端上报后抽检；上位机 OTA **必须**本地校验。

**大小限制**（建议）：日志 zip ≤ 500MB；exe ≤ 200MB；test_case bundle ≤ 50MB。

> 字段细节以总览 §5 为准；下表为后端实现索引。

### 6.1 认证 `/auth/*`

| 方法 | 路径 | 调用方 | 说明 |
|------|------|--------|------|
| POST | `/auth/login` | 上位机、网页 | body: username, password, **hostName**, deviceId?, stationKey? |
| POST | `/auth/refresh` | 上位机 | 二期 |
| POST | `/auth/logout` | 上位机、网页 | 作废 Token |
| GET | `/auth/me` | 网页 | 当前用户、角色、stationKeys |
| GET | `/admin/users` | 网页 admin | 分页列表 |
| POST | `/admin/users` | 网页 admin | 创建用户 |
| PUT | `/admin/users/{id}` | 网页 admin | 改角色、stationKeys、启用状态 |
| POST | `/admin/users/{id}/reset-password` | 网页 admin | 重置密码 |
| GET | `/admin/audit-logins` | 网页 admin | 登录审计分页 |

### 6.2 日志 `/logs/*`

| 方法 | 路径 | 调用方 | 说明 |
|------|------|--------|------|
| POST | `/logs/upload` | 上位机 | multipart: **factoryName**, deviceId, station, file(zip)；可选 sn, testResult, clientVersion |
| GET | `/logs` | 网页 | 筛选：**factoryName**, sn, station, deviceId, hostName, 时间范围 |
| GET | `/logs/{id}` | 网页 | 详情 + 解压后文件树 `files[]` |
| GET | `/logs/{id}/files/{path}` | 网页 | 单文件预览（`text/plain`，大文件分页） |
| GET | `/logs/{id}/download` | 网页 | 原 zip 签名 URL 或流式下载 |

**上传处理**：

1. 校验 `factoryName` 非空；未知代码可入库但标记 `unknown_factory` 告警（或拒绝，由配置定）。
2. 存 zip 至 OSS；写 `log_archives`（含 `factory_name`）。
3. **异步解压**：遍历 zip 内 `.txt` 等，写 `log_files` 表（`archive_id`、`relative_path`、`size`、`content_type`）；文本可抽 OSS 或 DB 存预览用对象。
4. 列表/详情 API 返回 `factoryName` 及 `factoryDisplayName`（查 `factories` 表）。

**过渡期**：`POST /logs/upload` 可允许匿名 + 仅 `deviceId`；接入登录后要求 Bearer，并与 login 的 deviceId/hostName 一致。

### 6.3 阈值 `/thresholds`

| 方法 | 路径 | 调用方 | 说明 |
|------|------|--------|------|
| GET | `/thresholds` | 上位机 | query: stationKey, productModel?；返回 version + items[{settingsKey,value}] |
| GET | `/admin/threshold-templates` | 网页 engineer+ | 模板列表 |
| GET | `/admin/threshold-templates/{id}` | 网页 | 详情 |
| POST | `/admin/threshold-templates` | 网页 | 新建草稿 |
| PUT | `/admin/threshold-templates/{id}` | 网页 | 编辑 |
| POST | `/admin/threshold-templates/{id}/publish` | 网页 | 发布 version |

**settingsKey** 必须与上位机 `上位机设置.ini` 键名完全一致（如 `BLE/LowRssi`）。

### 6.4 测试用例 `/test-cases`

| 方法 | 路径 | 调用方 | 说明 |
|------|------|--------|------|
| GET | `/test-cases/manifest` | 上位机 | bundleVersion, stationFlows, files[{path,version,sha256}] |
| GET | `/test-cases/bundle` | 上位机 | 返回 zip 流 |
| GET | `/test-cases/{name}` | 上位机 | 单 ini，可选 |
| GET | `/admin/test-cases/files` | 网页 | 文件树/列表 |
| PUT | `/admin/test-cases/files/{path}` | 网页 | 保存 ini 文本 |
| POST | `/admin/test-cases/publish` | 网页 | 打 bundle、升 bundleVersion |

### 6.5 上位机 OTA `/host-app`

| 方法 | 路径 | 调用方 | 说明 |
|------|------|--------|------|
| GET | `/host-app/check` | 上位机 | query: packageName, buildId, appVersion, stationKey, deviceId |
| GET | `/host-app/download/{buildId}` | 上位机 | exe 流或 302 到 OSS |
| GET | `/admin/host-app/versions` | 网页 admin | 版本列表 |
| POST | `/admin/host-app/versions` | 网页 admin | 上传元数据 + exe 到 OSS；sha256, releaseNotes, 灰度规则 |

灰度维度：`stationKeys[]`、`factory`、`deviceIds[]`、`forceUpgrade`（见总览 §5.4.6）。

### 6.6 统一发布 `/releases`（建议 P2）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/releases/check` | 上位机一次拉取：threshold / test-cases / host-app 是否有更新 |
| POST | `/admin/releases` | 网页：创建发布单，绑定各层 version |
| GET | `/admin/releases` | 发布历史 |

解决「有时更 ini、有时更 exe」的管理问题（见总览 §5.4.3 分工 + 本文 §8）。

### 6.7 基础数据 `/admin/meta`

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/admin/meta/stations` | 工站键与显示名字典 |
| GET | `/admin/meta/factories` | 工厂代码与中文名（如 `hz`→华庄、`byd`→比亚迪）；网页下拉与列表展示用 |
| GET | `/admin/meta/settings-keys` | 允许下发的 SETTINGS 键白名单 |
| GET | `/admin/devices` | 已登记 PC（hostName、产线、备注） |
| POST | `/admin/devices` | 登记 PC |

### 7.1 核心表

| `factories` | code（主键）, display_name, sort_order, enabled |
| `threshold_templates` | id, name, station_key, product_model, version, status(draft/published), published_at |
| `threshold_items` | id, template_id, settings_key, value |
| `test_case_files` | id, path, content_text, version, sha256, updated_at |
| `test_case_bundles` | id, bundle_version, object_key, manifest_json, published_at |
| `host_app_versions` | id, package_name, app_version, build_id, sha256, object_key, force_upgrade, gray_rules_json, release_notes, published_at |
| `releases` | id, release_id, note, threshold_version, bundle_version, host_build_id, gray_rules_json, created_at |

### 7.2 索引建议

- `log_archives(factory_name, created_at)`、`log_archives(station, created_at)`、`login_audit(host_name, created_at)`
- `host_app_versions(package_name, build_id)` 唯一

## 8. 发布与版本管理（后端）

三层版本 **独立递增**，由 **`releases`** 表可选绑定为一次「对外发布」：

| 层级 | 版本字段 | 存储 |
|------|----------|------|
| 阈值 | `threshold_templates.version` | DB |
| 用例 | `test_case_bundles.bundle_version` | DB + OSS zip |
| 上位机 exe | `host_app_versions.build_id` + `app_version` | OSS exe |

**原则**：改 ini/阈值 **不强制** 发 exe；`releases/check` 返回各层是否有更新及目标版本号。

## 9. 对象存储布局

```text
factory-tool/
  logs/{factory_name}/{yyyy}/{MM}/{deviceId}/{id}.zip
  logs/{factory_name}/{yyyy}/{MM}/{deviceId}/{id}/extracted/...
  test-cases/bundles/{bundleVersion}.zip
  host-app/{packageName}/{buildId}.exe
```

上传 exe/zip 后服务端计算 **sha256** 写入 DB；下载可经应用代理或 **短时签名 URL**。

## 10. 安全

- 全站 HTTPS；禁止明文 HTTP。
- 密码不入日志；Token 只存 hash（若持久化会话）。
- 上传 zip/exe：**大小上限**（如 zip 500MB、exe 200MB）、Content-Type 校验。
- 管理端接口统一 `@PreAuthorize` / 中间件校验 `admin` | `engineer`。
- 登录失败锁定（5 次 / 15 分钟，见总览 §5.6.4）。
- **CORS**：仅允许工厂云前端域名。
