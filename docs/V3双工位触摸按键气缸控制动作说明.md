# V3 双工位触摸按键测试 — PLC 气缸（按压）控制动作说明

本文档依据脚本 **《V3 双工位触摸按键自动测试》** 中的默认步骤计划与 PLC 参数推导整理。脚本通过汇川 PLC 的 **M 线圈** 驱动机台手指气缸下压/抬起；上位机只负责写 M 位，**位置到位**由 PLC 置位 `M(PLC_PositionReadyBase + 步骤索引)` 反馈，**按键完成**由 `M(PLC_KeyDoneM)` 只读握手。

---

## 1. PLC 控制方式与通信协议（接入自有代码必读）

### 1.1 用什么通信

- **链路**：以太网 **TCP/IP**。
- **协议**：脚本侧使用 **`InovancePLCModBus.InovancePLC`**（类名含 ModBus），对 PLC 执行 `Open(IpAddress, Port)` 后，以 **位读写** 方式访问 **M 区**（内部继电器线圈）。这与工业上常见的 **Modbus TCP** 用法一致。
- **默认端口**：`PLC_Port` 默认为 **502**（Modbus TCP 标准端口；脚本参数名 `PLC_Port`）。
- **PLC 型号参数**：`PLC_Series` 默认 **`H5U`**（汇川 H5U），在脚本里用于 `SetPLCType(PlcSeries)`，由厂商封装库按系列解析寻址与功能码。

### 1.2 脚本里如何「控制 PLC」

连接与断开（流程摘要）：

1. `PLC.SetPLCType(PLC_Series)` — 指定系列（如 H5U）。
2. `PLC.Open(PLC_IpAddress, PLC_Port)` — TCP 连接；脚本内带连接超时、发送/接收超时（参数 `PLC_TcpConnectTimeoutMs`、`PLC_TcpSendTimeoutMs`、`PLC_TcpReceiveTimeoutMs` 等）。
3. **写线圈**：`PLC.Write(address, coilValue)`，其中 `address` 为字符串形式的 **M 地址**，如 `"M200"`；`coilValue` 为 `bool`（脚本里由 `M200=1` / `M200=0` 解析为 true/false）。
4. **读线圈**：`PLC.Read(address, quantity)`，例如读 1 个位，返回 `bool[]`；用于轮询 **到位**、**按键完成**、**单点完成** 等反馈位。
5. 多条写命令之间可间隔 **`PLC_CommandGapMs`**（默认约 80 ms），避免 PLC 或总线来不及处理。
6. 流程结束或异常时 `PLC.Close()` 关闭连接。

**上位机禁止写入的位**：脚本明确禁止把 **`M(PLC_KeyDoneM)`** 当作输出命令写入（该位只能 **读**，由机台/PLC 侧置位表示「本步按键动作完成」）。

### 1.3 与「气缸动作」的对应关系

- 文档 **第 3 节「十五项气缸控制动作」** 所列 **`Mxxx=1` / `Mxxx=0`**，在实现上就是对上述地址调用 **`PLC.Write("Mxxx", true/false)`**。
- **握手**：写完下压命令后，脚本轮询 **`M(PLC_PositionReadyBase + stepIndex) == 1`** 表示位置到位；再轮询 **`M(PLC_KeyDoneM) == 1`** 表示可进入释放或下一步（具体顺序见原脚本 `RunStepActionAsync`）。

### 1.4 在你自己的 Qt/C++ 工程里怎么接

本仓库 **产测主程序当前未内置** 与脚本相同的 `InovancePLCModBus` DLL，若要把该能力并入 **C++**，常见做法如下（任选其一或组合）：

| 方案 | 说明 |
|------|------|
| **A. Qt Modbus TCP** | 在 `.pro` 中增加 `serialbus` 模块，使用 **`QModbusTcpClient`**。需按 **汇川 H5U 官方手册** 将 **`M` 继电器号** 换算为 Modbus **线圈起始地址 / 单元标识**（不同厂家对 M 的 Modbus 映射表不同，**不能**仅凭脚本里的 `M200` 字面量猜功能码与地址，必须以手册或汇川提供的地址计算工具为准）。 |
| **B. 原生 TCP 组帧** | 仅用 `QTcpSocket` 自行组 **Modbus TCP** 帧（如功能码 **0x05 写单线圈**、**0x01 读线圈**），同样需要 **M → 线圈地址** 的官方映射。 |
| **C. 复用厂商库** | 若现场已有与脚本一致的 **InovancePLCModBus**（.NET DLL），可通过 **C++/CLI**、**独立 C# 小进程 + 进程间通信**，或 **把控制逻辑留在现有脚本** 由 Qt `QProcess` 调起来，由退出码/文件回传结果。 |

**建议**：在实现前向汇川或集成商索取 **H5U 的 Modbus 地址表（M 区）** 或与脚本 DLL **等价的地址规则**，与本文档中的 **M200、M250、M260** 等逻辑编号对齐后再写 `Write/Read`。

### 1.5 与 BLE / 设备协议的关系（边界）

- **PLC 通信**只负责 **机台动作（气缸/手指到位）**。
- **按键事件、电容值**仍由 **BLE Shell（如 v3.exe）+ 设备协议** 完成；脚本里是两套通道并行：**TCP 502 ↔ PLC**，**串口 ↔ 产测 Shell**。并入 Qt 时通常也是：**一路 TCP 管 PLC**，**原有 `qfctp`/`qpb` 串口管设备**。

## 2. 工位与地址基准（默认参数）

| 项目 | 工位 1 | 工位 2 |
|------|--------|--------|
| 位置指令基底 `PLC_MBase` | M200 | M220 |
| 位置点范围（脚本说明） | M200～M209 | M220～M229 |
| 按键完成返回（只读）`PLC_KeyDoneM` | M210 | M230 |
| 测试完成输出 `PLC_TestDoneM` | M211 | M231 |
| Switch 气缸前推 `PLC_SwitchForwardM`（= MBase+12） | M212 | M232 |
| Switch 气缸复位 `PLC_SwitchResetM`（= MBase+13） | M213 | M233 |
| 异常大复位 `PLC_AbortResetM` | M240 | M241 |
| 位置到位返回基底 `PLC_PositionReadyBase` | M250 | M270 |
| 单点测试完成（共用）`PLC_StepDoneBase` | M260 | M280 |

普通按键步骤的 **到位索引** = 该步骤在计划中的顺序下标（0 起）；**switch** 步骤固定使用索引 **7**，对应到位位 `PLC_PositionReadyBase + 7`（工位 1 为 **M257**，工位 2 为 **M277**）。

## 3. 默认 `KeyTest_StepPlan` 中的 15 个气缸控制动作

以下按脚本内置 `BuildStepPlanExample()` / `BuildStation2DefaultPlan()` 的 **ActionCommands（下压）+ ReleaseCommands（抬起/复位）** 整理。**7 个物理键** 各 **下压、释放** 共 **14** 次线圈写入；**switch（旋钮）** 步骤为 **1** 组「前推 + 按压 + 释放」流程，合计 **15** 个气缸控制动作（与默认 8 步测试计划一一对应：前 7 步各 2 次，第 8 步为旋钮整组记作第 15 项）。

### 工位 1：十五项气缸控制动作（顺序与脚本默认计划一致）

1. **mode 下压**：`M200=1`  
2. **mode 抬起**：`M200=0`  
3. **program 下压**：`M201=1`  
4. **program 抬起**：`M201=0`  
5. **speed 下压**：`M202=1`  
6. **speed 抬起**：`M202=0`

8. **right 抬起**：`M203=0`

10. **start/pause 抬起**：`M204=0`  
11. **left 下压**：`M205=1`

## 4. 脚本中与气缸流程相关的其它 PLC 输出（非 15 项内）

这些属于 **流程握手 / 整机状态**，一般不记入「单键按压 15 动作」，但现场调试需知：

- **测试前 / 后**：`PLC_PrepareCommands`、`PLC_FinishCommands`（参数化，可为多条 `Mxxx=0/1`，以 `@@` 分隔）。
- **单点完成**：`M(PLC_StepDoneBase)=1`（可选脉冲后 `=0`，由 `PLC_StepDonePulseMs` 控制）。
- **整测完成**：`M(PLC_TestDoneM)=1`。
- **异常退出**：先尝试 `M(PLC_StepDoneBase)=1`，再 `M(PLC_AbortResetM)=1`。
- **DELAY= / COMMENT: / AFTER_READY:**：DSL 扩展；`AFTER_READY:` 段在 **位置到位** 后才执行，可用于二次小气缸或联锁。

## 5. 配置入口小结

- 步骤与气缸命令：`KeyTest_StepPlan` 或 `KeyTest_StepPlan_Station1` / `KeyTest_StepPlan_Station2`。
- 单条格式：`步骤名=>期望事件=>capId=>动作命令=>释放命令`，动作/释放内多条命令用 **`@@`** 拼接。
- 关闭自动 PLC 时设 `PLC_AutoTrigger=false`，则上述 M 输出由机台侧手动或其它逻辑承担。
- **自由工站 Qt 实现（`FREEWORK_TEST_LIST` 中 ID 81～95）**：与上表 15 项一一对应；**双工位**按当前窗口 **`getIndex()`（工位 1 / 2）** 解析地址，无需为两窗各写一套代码。
  - **`PLC/MBase`**：工位 1 的 M 起点，默认 **200**；工位 2 默认 **MBase + `PLC/MBaseStationStep`**（`PLC/MBaseStationStep` 默认 **20** → 220）。若某工位需单独指定，用 **`PLC/MBase_Station1`** / **`PLC/MBase_Station2`**（≥0 时优先于公式）。
  - **旋钮**：默认 **`resolvedPlcMBase()+12`**、**`+7`**；可配 **`PLC/SwitchForwardM_StationN`**、**`PLC/SwitchPressM_StationN`** 或全局 **`PLC/SwitchForwardM`** / **`PLC/SwitchPressM`**（≥0 时两工位共用绝对 M，慎用；未配则用基底+偏移）。
  - **连接校验读线圈**：**`PLC/ConnectVerifyM_StationN`** 优先，否则 **`PLC/ConnectVerifyM`**，再否则默认读 **`resolvedPlcMBase()`**（即本工位触摸区起点）。
  - **网络 / 从站 / 线圈偏移**：可选 **`PLC/IpAddress_StationN`**、**`PLC/Port_StationN`**、**`PLC/UnitId_StationN`**、**`PLC/MCoilAddressOffset_StationN`**；未配则回落到全局 **`PLC/IpAddress`** 等同名键。
