### （上位机可选）并联 CMW100 射频发包



勾选设置「并联 CMW100 射频刷包」即 `FreeInstrument/BleBrushCmwConcurrent=true`，并配置与 Wi‑Fi/BLE 工站一致的 `BlePer/CmwVisaAddress`、`BlePer/CmwWaveformFile`、`BlePer/CmwArbCycles` 等。自由工站在「停止接收与PER」步：**清空积包后同时启动等于 `BrushInstrument/PacketPhaseWaitMs` 的停止接收计时**，再并联跑 CMW；GPRF ARB **每个频点的 SCPI 顺序与 `docs/cmw100rx.txt` 对齐**（可先 `STAT OFF`、单次突发内 **`TRIGger:GPRF:GEN:ARB:MANual:EXECute` 再 `STAT ON;*OPC?`**）；默认 **`BlePer/CmwBurstPollArbScount=true`** 会轮询 `SOURce:GPRF:GEN:ARB:SCOunt?`（间隔见 `BlePer/CmwArbPollIntervalMs`，默认 100ms），达到设定周期后再用积包毫秒（PER 步与 `BrushInstrument/PacketPhaseWaitMs` 同源、`BlePer/CmwTriggerWaitMs` 等为其它场景的保持时间）补足。上位机日志会打出本机配置的波形路径（`BlePer/CmwWaveformFile`）、以及默认 **`BlePer/CmwQueryCurrentArbFile=true`** 时用 **`SOURce:GPRF:GEN:ARB:FILE?`** 读回的仪侧当前波形，另有（按 `BlePer/CmwVerboseArbPollLog`）SCOunt 轮询明细。若想退回「仅定时阻塞」可设 **`BlePer/CmwBurstPollArbScount=false`**。旧项 **`BlePer/CmwWaitArbScount=true`** 仍为「只轮询、不补足积包毫秒」语义。未勾选并联或未配 VISA 地址时跳过 CMW，仍按计时发停止接收。

配置项 **`FreeInstrument/BleBrushCmwOnStopPer`**（设置页「停止接收/PER步内再打一发GPRF」）为真时才在**每个** PER 步再打一发；若在「并联CMW播放Profile0」～「Profile5」六个独立步骤已对六路 profile 打过射频，可避免 PER1～PER6 重复射频，请将该项**取消勾选**。（六个播放步共用一次 GPRF ARB 参数配置初始化，不写 `STAT ON`/`TRIG`，出射频在每频点的 burst 路径中完成；参见 `freeWorkPrimeInstrumentCmwGprf`/`gInstrumentCmwGprfPrimed`。）
