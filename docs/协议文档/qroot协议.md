# 旧项目上位机已支持的协议-qroot


## 1. 数据包格式


### 1.1 帧结构

| 偏移 | 长度 | 字段 | 说明 |
|------|------|------|------|
| 0 | 2 | 帧头 | 固定 `0xAA 0x55` |
| 2 | 1 | 命令类型 | 见 §1.2 |
| 3 | 1 | 命令字 | 见 §2 |
| 4 | 1 | 参数长度 | 命令参数字节数 |
| 5 | 变长 | 命令参数 | 长度由参数长度决定 |
| 5+参数长度 | 1 | 校验和 | 见 §1.3 |


### 1.2 命令类型

| 值 | 名称 | 说明 |
|----|------|------|
| `0x00` | 请求 | 设备以肯定应答或否定应答回复 |
| `0x01` | 肯定应答 | 可带返回数据 |
| `0x02` | 否定应答 | — |
| `0x03` | 通知 | 查询下发或设备主动上报；查询类不要求再回肯定应答 |


### 1.3 校验和

对 **帧头至命令参数末字节**（不含校验和）逐字节求和，取低 8 位再按位取反：

```c
u8 checksum8(u8 *dat, u8 len) {
    u32 sum = 0;
    for (u8 i = 0; i < len; i++)
        sum += dat[i];
    return (u8)(~sum);
}
```


### 1.4 字节序说明

- 16 位字段在命令参数内按**小端**传输（低位字节在前）。
- **MAC 地址 6 字节**：线上顺序与显示字符串**反序**。  
  例：线上 `11 22 33 44 55 66` → 显示 `66:55:44:33:22:11`。

## 3. 命令说明

以下「上位机 → 设备」指上位机发出的帧；「设备 → 上位机」指设备回包或主动上报。  
命令表列为 **字段 | 值 | 说明**；帧公共字段用中文名（帧头、命令类型、命令字、参数长度、校验和），命令参数按语义或字节序号命名。同一参数字节多种取值时占一行：**值**列列出各取值，**说明**列对应说明，用顿号或分号分隔。
