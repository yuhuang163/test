# 业务功能模块概览

本项目是一个**产测上位机系统**，用于工厂产线对产品进行功能测试、校准、老化及数据上报。以下按业务维度介绍各模块。

---

## 一、协议通信层

负责上位机与外部设备（产品、治具、仪器、MES 系统）之间的数据交换。

| 模块 | 说明 |
|------|------|
| 产测协议 | 与待测产品通信的核心协议，用于下发测试指令、接收测试结果 |
| 治具协议 | 与生产治具通信，控制气动、电机等执行机构动作 |
| SCPI 协议 | 与程控仪器（电源、万用表、频谱仪等）通信的标准指令集 |
| Modbus 协议 | 与 PLC、传感器等工业现场设备通信 |
| AT 指令协议 | 与蓝牙/WiFi 模块通信的指令集 |
| ADB 协议 | 与 Android 设备通信的调试桥 |
| MES 协议 | 与工厂 MES 系统对接，上报生产数据、获取工单信息 |
| Bulk 协议 | 批量数据传输协议 |
| Shell 协议 | 远程 Shell 命令交互 |

## 二、业务功能模块

基于协议层实现的业务逻辑，对应具体的测试项或产品功能。

| 模块 | 说明 |
|------|------|
| BLE OTA 升级 | 通过蓝牙对产品进行固件远程升级，支持图片等资源升级 |
| CMW GPRF 射频测试 | 使用 CMW 综测仪进行蓝牙/WiFi 射频指标测试（PER、功率等） |
| PLC V3 治具控制 | V3 产品 PLC 治具控制，含触摸按键自动测试 |
| 三元组服务 | 读取并管理产品 SN、MAC、IMEI 等唯一标识信息 |

## 三、工站测试模块

每个工站对应产线一个独立测试工位，包含完整的测试流程与 UI。

| 工站 | 说明 |
|------|------|
| PCBA 板测 | 对 PCBA 裸板进行功能测试，是产线首道测试环节 |
| 屏幕测试 | 测试屏幕显示、触摸响应、色彩准确性 |
| 摄像头测试 | 测试摄像头模组成像质量、对焦功能 |
| 按键测试 | 测试物理按键手感、导通性、灵敏度 |
| 电机测试 | 测试电机运转状态、转速、工作电流 |
| 气压传感器测试与校准 | 测试气压传感器精度并进行校准 |
| IMU 校准 | 对加速度计、陀螺仪等惯性测量单元进行校准 |
| 静态电流测试 | 测试产品休眠/待机状态下的功耗 |
| 吸力测试 | 测试产品气动性能（吸力、负压等） |
| WiFi/BLE 射频测试 | 测试无线连接、信号强度、吞吐量等指标 |
| 老化测试 | 长时间通电老化，检验产品可靠性 |
| 自由工站 | 通用测试框架，可灵活配置测试项，适用非标场景 |

## 四、云平台服务

与云端工厂平台（FCTP/LuteOS）对接，实现数据上云与远程管理。所有子模块基于 `platform/cloud/` 目录实现。

| 子模块 | 说明 | 技术方案 | 功能介绍 |
|--------|------|----------|----------|
| **云平台客户端** | 与云端 API 通信的 HTTP 客户端基座，所有云服务的基础依赖 | HTTP/HTTPS 同步请求（QNetworkAccessManager + QEventLoop + 120s 超时）；JSON 数据序列化；请求头携带 Device-Id、Station-Key、APP-Version、Bearer Token；SSL 错误忽略处理；Base URL 默认为 `https://fctp.luteos.com`，支持配置切换 | 封装 GET/POST/Multipart 上传/文件下载等通用请求方法；统一解析服务端响应信封 `{ code, message, data }`；提供设备标识、工站标识、版本号等通用信息读取 |
| **云平台认证** | 用户登录鉴权与会话管理 | HTTP POST `/auth/login`；请求体为 JSON（username、password、hostName、deviceId、stationKey）；Token 持久化到 QSettings（FactoryCloud/Token + 过期时间） | 账号密码登录、记住密码、Token 自动续期判断、登出清除会话；支持多角色/多工站权限返回 |
| **测试数据上传** | 将产测结果上报至云端工厂平台 | HTTP POST `/test-records`；JSON 请求体包含工厂、工站、SN、测试结果、测试项明细列表；支持同步/异步两种模式（异步走 QtConcurrent::run 不阻塞 UI） | 按条上报产测数据（SN、各测试项值、上下限、单位、结果）；支持异步上传不阻塞测试流程；功能开关 `FactoryCloud/Feature/TestDataUpload` 控制启停 |
| **测试记录本地存储** | 本地 SQLite 缓存测试记录，防止断网丢数据 | 本地 SQLite 数据库（`test_pass_data.db`）；两张表：`mes_test_pass`（通用通过记录）+ 按工站名的独立明细表（SN + 测试项维度，`INSERT OR REPLACE`）；线程安全（QMutex 保护） | 每次测试通过后自动写入本地库；测试项数据格式 `\|name:val:max:min:std:unit:result\|` 解析；自动建表与 Schema 迁移（旧宽表格式检测并重建）；为后续补传/追溯提供数据基础 |
| **日志上传** | 将上位机运行日志压缩后上传云端，便于远程排查 | 两步上传策略：优先 Qt QHttpMultiPart，失败则回退到 curl.exe；压缩使用 PowerShell `Compress-Archive`（120s 超时）；Multipart 表单字段含 deviceId、station、factoryName、SN、testResult 等 | 自动扫描日志目录（`所有log/`）压缩为 ZIP；上传至 `POST /logs/upload`；含 Sn 与测试结果等现场信息辅助定位；上传超时 300s；功能开关 `FactoryCloud/Feature/LogUpload` |
| **远程升级** | 支持上位机自身 OTA 远程升级 | HTTP GET 检查更新（`/host-app/check`）、下载安装包（`/host-app/download/{buildId}`）；SHA-256 完整性校验；下载后通过批处理脚本（`delete_self.bat`）实现原地替换：备份旧 exe → 替换新 exe → 启动新进程 → 清理备份；支持断点式版本选择 | 检查云端是否有新版本；强制升级/可选升级两种模式；下载后自动校验 SHA-256；静默替换重启；本地上位机版本注册上传（`POST /host-app/upload`）；版本选择器对话框支持手动选版；功能开关 `FactoryCloud/Feature/HostOtaCheck` |
| **测试用例同步** | 从云端同步最新测试用例与配置 | HTTP GET 获取清单（`/test-cases/manifest`）→ 下载 ZIP 包（`/test-cases/bundle`）→ SHA-256 逐文件校验 → 本地备份旧用例 → 解压部署；上传走 Multipart（`POST /test-cases/upload`）；开机异步同步（QtConcurrent::run 不阻塞启动） | 开机自动同步云端最新测试用例；本地用例打包上传云端；清单版本比对，相同版本跳过；部署前自动备份旧用例到 `.backup/`；校验失败自动回滚；功能开关 `FactoryCloud/Feature/TestCaseSync` |

## 七、配置与设置

上位机的配置管理中心，采用**多标签页设置界面 + 可视化测试流程编排 + 表驱动配置绑定**架构，覆盖全系统所有可调参数。

| 模块 | 说明 |
|------|------|
| **多标签全局设置界面** | 提供"上位机设置"主窗口，包含多个配置标签页，覆盖所有工站的公共参数与专属参数 |
| **工站切换** | 支持 13 种工站类型（调试/电机/静态电流/IMU/屏幕/摄像头/信号/老化/压感/板测/按键/吸力/自由工站），切换后自动重启加载新配置 |
| **MES 对接配置** | 支持立讯、欣旺达、华勤、伟克森、雅达明、BYD、华庄等多工厂 MES 系统，动态显示对应字段（MES 地址、工站名、制程、机种、工单、人员、线体等） |
| **通信协议选择** | 支持 qpb（产测 PB 协议）、qfctp（FCTP 协议）、qaiot（AIOT TLV 协议）、qroot（吸奶器 PCBA 蓝牙透传）四种协议栈切换 |
| **产品差异化开关** | 约 25 项产品特性开关（是否写 subPID/SKUID、IMU 是否需要晃起、蓝牙图片传输、音频测试、船运电流、舵机启动、灯光测试等），切产品时自动恢复默认值 |
| **SN 规则配置** | 配置产品 SN 的正则表达式校验规则 |
| **1拖多布局** | 配置同时管理多个治具的网格布局（行列数），适配多工位并行测试 |
| **测试详细配置** | 分标签页配置各工站专属参数，包括：老化模式与时长、按键 ID 与电容触摸阈值、三元组（环境/URL/账号/SKU）、云平台（环境/登录/功能开关）、PLC Modbus 参数（IP/端口/线圈偏移/双工位）、CMW100 射频参数（PER 发包数/门限/时序）、电流阈值（货运动态/充电/工作/静态/音频）、IMU 校准参数（时长/收敛阈值/轴限值）、压感参数（模式/ADC 阈值/摆幅限值）、治具预期值、摄像头参数（ROI/间隔/脏污检测延迟）、产品版本比对项、自由工站外设状态预期值等 |
| **表驱动配置绑定** | 通过 QSettingBindings 模块，以声明式表格将 200+ 个 UI 控件与 INI 配置键一一绑定，消除重复的 load/save 代码，统一维护默认值与工具提示 |
| **测试流程编排** | 可视化拖拽式测试流程编辑器，工程师无需编码即可设计每站测试顺序：支持站流程管理（增删改查/复制）、测试块拖拽排序、批量添加测试用例、空白块占位、右键删除、单步失败停止开关、未保存修改提示 |
| **测试用例编辑** | 双击测试块弹出编辑对话框，支持配置通信通道（产品蓝牙/产品串口/Dongle/云/治具/Modbus/SCPI）、指令选择与参数编辑、前后延时与超时、合格判定门限（单字段/多字段/比对方式）、特殊步骤挂钩（PLC 旋钮/按键序列/CMW 射频测试/SN 写入等工程级功能） |
| **治具串口面板** | 治具串口连接管理控件，下发治具指令（IMU 状态/启动/休眠/PCBA 数据等），支持多机同步 |
| **本地持久化** | 所有配置通过 QSettings 写入 INI 文件（团队配置入库 + 本机常改项自动路由到 local.ini） |

## 十、自定义配置模块

提供应用级别的全局类型定义、通用工具函数与配置管理能力。

| 模块 | 说明 |
|------|------|
| 全局类型与数据结构 | 定义 MES 上报数据包结构体（MesPacketData），包含工厂、工站、SN、测试项、结果、上下限、单位等字段，统一各工站与 MES 对接的数据格式 |
| 双文件 INI 配置管理 | 采用"入库配置 + 本地覆盖"双文件策略——团队公共配置写入"上位机设置.ini"（入库），串口号、窗口大小、当前工站、WiFi 名称等本机常改项自动路由到"上位机设置.local.ini"（gitignore），避免开发人员互相干扰 |
| 配置读写封装 | SettingsManager 单例封装，提供 value/setValue/beginGroup/childGroups/remove 等接口，自动判断键名归属文件，上层无需关心存储细节 |

| 各工站版本号集中管理 | 在 AbIni.h 中集中定义所有工站的版本宏（DEBUG_VER、CAMERA_VER、PCBA_VER、IMU_VER、SCREEN_VER 等），统一版本追溯 |

## 模块关系示意图

```
仪器 / 治具 / 产品  ↔  协议层  →  业务层  →  工站层  →  云平台  →  MES / 云端工厂
```

各模块通过协议层与硬件交互，经过业务逻辑处理后在工站中执行测试，最终将测试结果上报云端或 MES。
