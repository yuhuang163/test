# 协议适配改造方案（分阶段执行）

> 目标：把 `agreement` 从“按协议类型堆代码”逐步改为“传输复用 + 协议编解码可替换”的结构。  
> 原则：小步落地、每步可编译、业务行为不突变。

---

## 阶段规划

### 阶段 0：目录职责约定（不改行为）

| 分层 | 目录 | 说明 |
|---|---|---|
| 传输层 | `agreement/qtransport/` | 通道与收包工具 |
| 协议层 | `agreement/qProtocol/` | qpb / qfctp / qaiot + manager |
| 仪器层 | `agreement/qusb`、`agreement/scpi`、`qjig`、`qmomcozy` | 串口/VISA/治具/产品仪器 |
| 主机侧 | `agreement/qadb`、`agreement/qshell` | 进程命令通道 |
| 云侧 | `agreement/qmes`、`agreement/qtuple` | HTTP/HTTPS |

---

### 阶段 1：串口通道 + Dongle 收包统一（已完成）

- 新增 `qtransport/qchannel.*`，协议发送统一走 `QSerialChannel::write`。
- 新增 `qtransport/qserialportreader.*`，Dongle 收包由 Reader 防抖。
- `Qpb/Qfctp/Qaiot` 写口已接入 `QSerialChannel`。
- `test_base` 中 Dongle 收包已改为 `QSerialPortReader`。

---

### 阶段 2：仪器/PLC 复用传输（已完成第一版）

#### 2.1 本次落地（已完成）

- 新增 `agreement/qtransport/qmodbus_pdu.h/.cpp`：
  - `buildReadCoilsRequestPdu`
  - `buildWriteSingleCoilRequestPdu`
  - `appendUint16Be/readUint16Be`
  - `crc16ModbusRtuBigEndian`
- `agreement/qplc/inovance_h5u_modbus_tcp.cpp` 已改为使用 `QModbusPdu` 构 PDU。
- `agreement/qusb/qusb.cpp` 的 RTU CRC 已复用 `QModbusPdu::crc16ModbusRtuBigEndian`。

#### 2.2 本次补充（已完成）

1. USB/jig/product 收包已迁移 `QSerialPortReader`（`test_base`）。
2. Modbus PDU 校验：`isExceptionResponse`、`functionCodeMatches`、`validateRtuFrame`。
3. 异常码文字化：`exceptionCodeText`、`formatExceptionMessage`（`qplc` 已接入）。
4. 仪器写口统一：`qusb`、`qat`、`qjig` 发送改走 `QSerialChannel::write`。

#### 2.3 本次补充（已完成）

- `parseReadCoilsPdu`：TCP/PDU 读线圈响应解析，`qplc` 已接入。
- `qjig` 已移除 DAM-3158 采集卡相关逻辑；吸力工站默认走 Pico 传感器。
- `fixture_uart` 写口改走 `QSerialChannel::write`。

#### 2.4 后续可选

- `fixture_uart` 收包防抖可迁移 `QSerialPortReader`（当前仍用独立 Timer）。

---

### 阶段 3：进程类合并（已完成第一版）

- 已新增 `agreement/qtransport/qprocesschannel.*`（命令队列、endMark、timeout）。
- `Qshell` 已接入 `QProcessChannel`，保留 PowerShell 启动参数与初始化命令差异。
- `Qadb` 已接入 `QProcessChannel`，保留 `adb shell` 与按键监控（`keyProcess`）差异。
- `QProcessChannel::setRestartOnTimeout(true)` 已用于 `Qadb`（超时后 kill 并重启 adb shell，对齐旧行为）。

---

### 阶段 4：编解码插件化（长期）

```cpp
class IProtocolCodec {
public:
    virtual QByteArray encodeRequest(DeviceCmd cmd, const QVariant& data) = 0;
    virtual void feedBytes(const QByteArray& chunk) = 0;
};
```

- `QProtocolManager` 最终持有 `IProtocolCodec*`（而非具体 `Qpb/Qfctp/Qaiot` 指针）。
- 该阶段涉及面大，不在当前迭代强推。
